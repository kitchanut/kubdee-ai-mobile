# Shopee Post Flow

เอกสารนี้สรุป flow ที่ต้องยึดเป็นหลักสำหรับ automation โพสต์วิดีโอ Shopee โดยเฉพาะขั้นตอนแนบสินค้า เพราะ Shopee แต่ละเวอร์ชัน/แต่ละเครื่อง expose accessibility node ไม่เหมือนกัน

## เป้าหมาย

โพสต์วิดีโอจาก Kubdee AI ไป Shopee Video พร้อมแนบสินค้าให้ถูกต้อง โดยใช้ลิงก์สินค้าเป็นทางหลัก และ fallback ไปค้นหาด้วยชื่อสินค้าเมื่อ path ลิงก์ใช้ไม่ได้

## Flow หลัก

1. เปิด Shopee ใหม่ก่อนเริ่มงาน
2. ไปที่ `ฉัน`
3. เปิด `โปรแกรม Affiliate`
4. ไปที่ `บัญชีผู้ใช้`
5. เปิดหน้า `Shopee Video`
6. กด `โพสต์วิดีโอ`
7. เปิดคลังภาพ/วิดีโอ
8. เลือกวิดีโอที่ Kubdee เตรียมไว้ล่าสุด
9. กดถัดไปจาก preview
10. กดถัดไปจาก editor
11. กรอกแคปชั่น
12. แนบสินค้า
13. ปิดตัวเลือกที่ไม่ต้องการ เช่น reuse/เผยแพร่ต่อ ถ้าหาได้
14. เปิดป้ายกำกับ AI ถ้าหาได้
15. กดโพสต์
16. รอ Shopee รับคำสั่ง แล้วกลับ Kubdee AI

## แนบสินค้าด้วยลิงก์

ลำดับนี้สำคัญมาก และเป็น flow ที่เคยทำได้ในช่วงประมาณ `0.2.44`:

1. กดแถว `แตะเพื่อเพิ่มสินค้า`
2. ถ้าเมนูไม่เปิด ให้ใช้ fallback แตะแถวเพิ่มสินค้า
3. ในหน้า `เพิ่มสินค้า` ให้กดไอคอนโซ่มุมขวาบน
4. เข้าไปหน้า `กรอกลิงก์สินค้า`
5. วาง/กรอก URL สินค้าในช่องลิงก์
6. verify ว่าลิงก์อยู่ในช่องจริง
7. กด `นำเข้า`
8. รอรายการสินค้าจากลิงก์โหลด
9. กด `เลือกทั้งหมด`
10. กด `เพิ่ม`
11. รอจนกลับหน้า editor หรือเห็นว่าสินค้าถูกแนบแล้ว

ห้ามกด `เพิ่ม` ทันทีหลังวางลิงก์ เพราะ `เพิ่ม` ที่อยู่ล่างหน้าจอเป็นปุ่มคนละขั้นกับ `นำเข้า` และจะทำให้ลิงก์ไม่ถูก import

## จุดที่ต้องระวัง

- ปุ่ม `นำเข้า` อยู่ใต้ช่องลิงก์ในกล่อง `กรอกลิงก์สินค้า`
- ปุ่ม `เพิ่ม` อยู่ในขั้นหลังจาก import แล้วและเลือกสินค้าแล้ว
- ถ้า log ขึ้นว่า `กดปุ่มยืนยันลิงก์ 'เพิ่ม'` แปลว่ากดผิด flow
- หลังวางลิงก์ควรมี log ประมาณ `verify=ok` ก่อนกด `นำเข้า`
- ถ้า verify ไม่ได้ว่าลิงก์อยู่ในช่อง ห้ามกด `นำเข้า`
- ถ้ากด `นำเข้า` แล้ว Shopee กลับไปหน้า editor พร้อม URL อยู่ในแคปชั่น แปลว่า import ไม่สำเร็จ
- resource id ของ Shopee อาจต่างกันระหว่างเครื่อง/เวอร์ชัน จึงไม่ควรพึ่ง resource id อย่างเดียว
- icon โซ่บางเครื่องไม่ expose เป็น `ImageView` หรือไม่มี `contentDescription`
- overlay log อาจบัง header/ปุ่มได้ จังหวะค้นหา icon หรือแตะ header ต้องซ่อน overlay ก่อน

## Fallback เมื่อแนบจากลิงก์ไม่ได้

ถ้า path ลิงก์ไม่สำเร็จ:

1. กลับไปหน้า editor ถ้าหลุดออกจากเมนู
2. เปิดเมนูเพิ่มสินค้าใหม่
3. ใช้ชื่อสินค้าเพื่อค้นหา
4. เลือกสินค้าที่เจอ
5. กด `เพิ่ม` หรือ `เสร็จ`
6. รอจนกลับหน้า editor
7. โพสต์ต่อ

## Log ที่ควรอ่านง่าย

ควรมี log แยกสถานะเหล่านี้:

- เปิดเมนูเพิ่มสินค้าได้หรือไม่
- เปิดหน้ากรอกลิงก์ด้วยวิธีใด เช่น text, icon, หรือ header fallback
- กรอกลิงก์สำเร็จหรือไม่
- verify เห็นลิงก์ในช่องหรือไม่
- กด `นำเข้า` ด้วย text หรือ fallback ใต้ช่องลิงก์
- import แล้วเจอรายการสินค้าหรือไม่
- กด `เลือกทั้งหมด` แล้วหรือไม่
- กด `เพิ่ม` แล้วหรือไม่
- กลับ editor พร้อมสินค้าแนบแล้วหรือไม่
- ถ้า fail ให้บอกว่า fail ที่ขั้นไหน และจะ fallback ไปค้นหาชื่อหรือไม่

## ตัวอย่าง log ที่ถูกต้อง

```text
กรอกลิงก์สินค้า: setText=ok verify=ok bounds=...
กรอกลิงก์สินค้า: เห็นลิงก์ในช่องแล้ว รอ 1.8 วิ ก่อนกดนำเข้า
กรอกลิงก์สินค้า: กดนำเข้าแล้ว
กรอกลิงก์สินค้า: รายการสินค้าจากลิงก์โหลดแล้ว
กรอกลิงก์สินค้า: กดเลือกทั้งหมดแล้ว
กรอกลิงก์สินค้า: กดเพิ่มสินค้าจากลิงก์แล้ว
แนบสินค้าด้วยลิงก์แล้ว
```

## ตัวอย่าง log ที่ผิด flow

```text
กรอกลิงก์สินค้า: setText=ok verify=ok bounds=...
กรอกลิงก์สินค้า: กดปุ่มยืนยันลิงก์ 'เพิ่ม' ที่ ...
กรอกลิงก์สินค้า: เจอหน้า editor แทนช่องลิงก์
```

กรณีนี้คือกด `เพิ่ม` เร็วเกินไปหรือกดผิดปุ่ม ต้องกลับไปแก้ให้กด `นำเข้า` ก่อน

## Regression Checklist

- หลังวางลิงก์ ต้องไม่กด `เพิ่ม` ทันที
- ต้องกด `นำเข้า` ก่อนเสมอ
- หลัง `นำเข้า` ต้องรอรายการสินค้า
- ต้องกด `เลือกทั้งหมด`
- ต้องกด `เพิ่ม` หลังเลือกสินค้าแล้วเท่านั้น
- ถ้าลิงก์ fail ต้อง fallback ไปค้นหาด้วยชื่อได้
- สุดท้ายต้องกลับ editor และกดโพสต์ได้
